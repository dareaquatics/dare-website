    1. Introduction

    The DARE Aquatics Website Editing SDK is a comprehensive toolkit designed to streamline the management and updating of the DARE Aquatics website. This SDK offers a suite of Python scripts and a graphical user interface (GUI) application that automate content synchronization, facilitate easy editing, and manage version control through GitHub integration.

    Key Features

    graph TD A[DARE Aquatics SDK] --> B[Automated Sync] A --> C[GUI Editor] A --> D[Version Control] B --> E[News Sync] B --> F[Calendar Sync] C --> G[Text Editing] C --> H[Image Management] C --> I[Link Editing] D --> J[GitHub Integration] D --> K[Commit History]

    2. System Requirements

    • Python 3.9 or higher
    • Git 2.25 or higher
    • Operating System: Windows 10+, macOS 10.14+, or Linux (Ubuntu 18.04+, CentOS 7+)
    • Minimum 4GB RAM
    • 100MB free disk space
    • Internet connection for GitHub synchronization and TeamUnify access

    3. Installation

    1. Clone the DARE Aquatics website repository: 
      git clone https://github.com/dareaquatics/dare-website.git
cd dare-website
    2. Set up a virtual environment (recommended): 
      python -m venv venv
source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
    3. Install required Python packages: 
      pip install -r requirements.txt
    4. Set up GitHub personal access token:
      • Create a personal access token on GitHub with repo permissions.
      • Set the token as an environment variable:
        • On Unix-based systems: 
          export PAT_TOKEN=your_token_here
        • On Windows: 
          setx PAT_TOKEN your_token_here
    5. Verify installation: 
      python -c "import requests, beautifulsoup4, git, PIL, cloudscraper, colorlog, tqdm, ics, pytz; print('All dependencies installed successfully')"

    5. GitHub Actions Workflows

    5.1 autosync-run.yaml

    This workflow is responsible for automating the news update process.

    Key Features:

    • Runs on both manual trigger and a scheduled basis
    • Executes the autosync.py script
    • Handles dependency installation
    • Uses a GitHub secret for authentication

    Workflow Structure:

    name: Update News

on:
  workflow_dispatch:
    inputs:
      force_update:
        description: 'Force update the news'
        required: false
        default: 'false'
  schedule:
    - cron: '0 */2 * * *'
    - cron: '30 */2 * * *'

jobs:
  update-news:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install cloudscraper beautifulsoup4 gitpython colorlog tqdm

      - name: Run the script
        env:
          PAT_TOKEN: ${{ secrets.PAT_TOKEN }}
        run: |
          python autosync.py

    Usage:

    • The workflow runs automatically every 2 hours at 0 and 30 minutes past the hour.
    • It can also be triggered manually from the GitHub Actions tab.

    5.2 autosync_calendar.yaml

    This workflow automates the calendar event update process.

    Key Features:

    • Runs on both manual trigger and a daily schedule
    • Executes the autosync_calendar.py script
    • Handles dependency installation
    • Uses a GitHub secret for authentication

    Workflow Structure:

    name: Update Calendar

on:
  workflow_dispatch:
    inputs:
      force_update:
        description: 'Force update the calendar'
        required: false
        default: 'false'
  schedule:
    - cron: '0 0 * * *' # Runs daily at midnight

jobs:
  update-calendar:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install ics gitpython colorlog requests tqdm pytz

      - name: Set up Git
        run: |
          git config --global user.name "github-actions[bot]"
          git config --global user.email "github-actions[bot]@users.noreply.github.com"
          git remote set-url origin https://x-access-token:${{ secrets.PAT_TOKEN }}@github.com/dareaquatics/dare-website.git

      - name: Run the script
        env:
          PAT_TOKEN: ${{ secrets.PAT_TOKEN }}
        run: |
          python autosync_calendar.py

    Usage:

    • The workflow runs automatically every day at midnight.
    • It can also be triggered manually from the GitHub Actions tab.

    6. Usage Guide

    6.1 Automated News Synchronization

    1. Ensure your environment is set up correctly with the required dependencies and GitHub token.
    2. Run the autosync.py script: 
      python autosync.py
    3. The script will automatically:
      • Clone or update the local repository
      • Fetch news from TeamUnify
      • Update the news HTML file
      • Commit and push changes to GitHub

    For automated regular updates, consider setting up a cron job (Linux/macOS) or Task Scheduler (Windows) to run the script periodically.

    6.2 Automated Calendar Synchronization

    1. Verify that your environment is properly configured.
    2. Execute the autosync_calendar.py script: 
      python autosync_calendar.py
    3. The script will:
      • Fetch events from the specified ICS file
      • Update the calendar HTML file
      • Commit and push changes to GitHub

    Similar to news synchronization, you can automate this process using cron jobs or Task Scheduler.

    6.3 Using the GUI Editor

    1. Launch the editor application: 
      python editor.py
    2. Select an HTML file to edit from the dropdown menu.
    3. Use the various tabs to edit text, manage images, or modify links.
    4. After making changes, use the "Commit Changes" button to push your updates to GitHub.

    8. Configuration

    8.1 Environment Variables

    • PAT_TOKEN: GitHub Personal Access Token for repository access.

    8.2 Constants (autosync.py and autosync_calendar.py)

    • GITHUB_REPO: URL of the GitHub repository
    • NEWS_URL: URL of the TeamUnify news page
    • REPO_NAME: Name of the local repository
    • NEWS_HTML_FILE: Name of the HTML file to update for news
    • ICS_URL: URL of the ICS file containing events
    • EVENTS_HTML_FILE: Name of the HTML file to update for events

    9. Best Practices

    9.1 Version Control

    • Always pull the latest changes before making edits to ensure you're working with the most up-to-date content.
    • Use meaningful and descriptive commit messages.
    • Regularly review the commit history to track changes.

    9.2 Content Management

    • Maintain consistent formatting when editing HTML content.
    • Use the provided styling help in the GUI for proper HTML tag usage.
    • Verify links are correct and functioning after editing.

    9.3 Image Handling

    • Use descriptive file names for uploaded images.
    • Optimize images for web before uploading to maintain site performance.
    • Be cautious when deleting images, as they may be used in multiple places on the website.

    9.4 Automation

    • Set up automated runs of autosync.py and autosync_calendar.py at appropriate intervals.
    • Monitor logs regularly to catch and address any synchronization issues.

    9.5 Security

    • Keep your GitHub Personal Access Token secure and rotate it periodically.
    • Use a dedicated service account for automated processes if possible.

    9.6 Testing

    • After making changes, always preview the website to ensure updates appear as expected.
    • Test on multiple browsers and devices to ensure compatibility.

    10. Troubleshooting

    Common Issues and Solutions

    1. Git-related errors:

    • Ensure Git is properly installed and configured.
    • Check if the PAT_TOKEN environment variable is set correctly.

    2. Authentication issues:

    • Verify that your GitHub token is valid and has the necessary permissions.
    • Regenerate the token if needed and update the environment variable.

    3. GUI fails to launch:

    • Confirm all required dependencies are installed (pip install -r requirements.txt).
    • Check Python version compatibility (3.9+).

    4. Synchronization failures:

    • Verify internet connectivity.
    • Ensure the TeamUnify news URL and ICS URL are correct and accessible.
    • Check log files for specific error messages.

    5. HTML update issues:

    • Verify write permissions for the HTML files in the local repository.
    • Ensure the correct HTML file names are specified in the configuration.

    Logging

    Both autosync.py and autosync_calendar.py use Python's logging module to record operations and errors. Check the console output or log files for detailed information about script execution.

    The GUI application (editor.py) displays logs in a dedicated console within the interface.

    Debugging

    For advanced debugging:

    1. Set the log level to DEBUG in the scripts: 
      logging.basicConfig(level=logging.DEBUG)
    2. Use Python's debugger (pdb) by adding breakpoints: 
      import pdb; pdb.set_trace()
    3. For GUI issues, use print statements or logging to track the flow of execution.

    15.1 Customizing the HTML Generation

    Both autosync.py and autosync_calendar.py use the generate_html() function to create HTML content. You can customize this function to change the layout or styling of the generated content.

    Example of customizing the news HTML generation in autosync.py:

    def generate_html(news_items):
    news_html = '<div class="news-container">'
    for item in news_items:
        news_html += f'''
        <article class="news-item">
            <h2 class="news-title">{item["title"]}</h2>
            <p class="news-meta">By {item["author"]} on {item["date"]}</p>
            <div class="news-content">{item["summary"]}</div>
        </article>
        '''
    news_html += '</div>'
    return news_html

    15.3 Implementing Custom Logging

    While the SDK uses Python's built-in logging module, you might want to implement a custom logging solution for specific needs. Here's an example of how to create a custom logger:

    import logging
from logging.handlers import RotatingFileHandler

def setup_custom_logger(name):
    formatter = logging.Formatter(fmt='%(asctime)s - %(levelname)s - %(module)s - %(message)s')

    handler = RotatingFileHandler('dare_sdk.log', maxBytes=10*1024*1024, backupCount=5)
    handler.setFormatter(formatter)

    logger = logging.getLogger(name)
    logger.setLevel(logging.DEBUG)
    logger.addHandler(handler)

    return logger

# Usage
logger = setup_custom_logger('dare_sdk')
logger.debug("This is a debug message")
logger.info("This is an info message")

    15.4 Optimizing Performance

    For larger websites or frequent updates, you might need to optimize the performance of the synchronization scripts. Here are some tips:

    • Use async operations for network requests to fetch news and events concurrently.
    • Implement caching to avoid unnecessary API calls or file operations.
    • Use multiprocessing for CPU-intensive tasks like HTML generation for large datasets.

    Example of using async operations with aiohttp:

    import aiohttp
import asyncio

async def fetch_news_async(url):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()

async def main():
    news_html = await fetch_news_async(NEWS_URL)
    # Process the news_html

asyncio.run(main())
